## 問題1: サブグラフ優先吸着問題

### 概要

ユーザーが個別ノード（PHASE1, PHASE2等）をクリックしたつもりでも、大きなサブグラフ（SCOPE等）に吸着してしまう。

### 期待動作 vs 実際の動作

| 項目 | 期待動作 | 実際の動作 |
|------|----------|------------|
| クリック位置 | PHASE1ノード付近（目視で近い） | PHASE1ノード付近（同じ） |
| 検出されるノード | PHASE1 (個別ノード) | SCOPE (サブグラフ) |
| 吸着先 | PHASE1の中心座標 | SCOPEの中心座標 |
| ハイライト | PHASE1が緑色に光る | SCOPEが緑色に光る |
| エッジ接続先 | PHASE1 | SCOPE |

### 実際のコンソールログ（抜粋）

```
📍 Click at (1257.3, 1898.1), screen radius: 1000px, adjusted: 666.7px
🎯 Top 5 closest nodes:
  1. PHASE3: 2547.9px - center:(1603.7, 4422.4) bbox:(1306.9, 3645.0) 593.7x1554.7
  2. PHASE2: 628.6px - center:(977.6, 2461.0) bbox:(8.0, 1348.0) 1939.1x2226.0
  3. PHASE1: 1104.5px - center:(1599.7, 848.0) bbox:(1300.7, 419.0) 598.0x858.0
  4. SCOPE: 299.5px - center:(1371.7, 1911.5) bbox:(937.2, 1510.0) 869.0x803.0
🎯 ノード吸着: SCOPE (距離: 299.5px)
```

### ユーザーレポート

> サブグラフは緑ハイライトになりましたが、接続したいノード上では緑ハイライトになりませんでした。とりあえず緑ハイライトされたサブグラフへ接続するためにエッジラベル入力画面へtestと入力してOKとしましたが、そのエッジとエッジラベルは表示されませんでした。

### 技術的な原因分析

1. **距離計算**:
   - PHASE1の中心: (1599.7, 848.0)
   - SCOPE（サブグラフ）の中心: (1371.7, 1911.5)
   - クリック位置: (1257.3, 1898.1)

2. **ユークリッド距離**:
   - PHASE1まで: 1104.5px（遠い）
   - SCOPEまで: 299.5px（近い）← **これが選ばれる**

3. **なぜSCOPEが近い？**:
   - SCOPEは大きなサブグラフで、bbox = (937.2, 1510.0) 869.0x803.0
   - クリック位置(1257.3, 1898.1)はSCOPE内部またはその近くにある
   - 中心距離で判定しているため、大きな要素の中心が近くなる

### 望ましい動作

1. **優先度判定の追加**:
   - 個別ノード（`data-et="node"`）を優先
   - サブグラフ（`data-et="cluster"`）は二次的な候補

2. **または、サイズ考慮**:
   - bboxが小さいノードを優先
   - 例: 面積が閾値以下のノードを優先

3. **または、境界距離**:
   - 中心距離ではなく、bbox境界までの距離で判定

## 問題2: エッジ非表示問題

### 概要

エッジ作成処理は成功しているが、SVG画面上にエッジが表示されない。

### 期待動作 vs 実際の動作

| 項目 | 期待動作 | 実際の動作 |
|------|----------|------------|
| エッジ作成 | ✅ 成功 | ✅ 成功（ログ確認済み） |
| SVG追加 | ✅ appendChild実行 | ✅ appendChild実行（ログ確認済み） |
| 画面表示 | ✅ 青い破線が見える | ❌ 何も見えない |
| ラベル表示 | ✅ "test"が見える | ❌ 何も見えない |

### コンソールログ（成功している）

```
🎨 drawEdgeToSvg called: {id: "manual-edge-1234567890", ...}
✅ SVG element found: <svg ...>
✅ Edge manual-edge-1234567890 added to SVG: {
  start: "(1371.7, 1911.5)",
  end: "(1468.7, 1626.8)",
  label: "test"
}
```

### エッジのSVG構造（生成されているはず）

```svg
<g class="manual-edge" data-id="manual-edge-1234567890" data-et="manual-edge">
  <line
    x1="1371.7"
    y1="1911.5"
    x2="1468.7"
    y2="1626.8"
    stroke="#007bff"
    stroke-width="5"
    stroke-dasharray="5,5"
    marker-end="url(#manual-edge-arrowhead)"
    pointer-events="none"
  ></line>

  <text
    class="manual-edge-label"
    x="1420.2"
    y="1769.15"
    text-anchor="middle"
  >test</text>
</g>
```

### 考えられる原因

1. **Z-index問題**:
   - 他の要素の下に隠れている？
   - SVG内ではz-indexは使えず、DOM順序で決まる

2. **ViewBox範囲外**:
   - 座標が現在のviewBox範囲外にある？
   - ズーム（150%）やスクロール位置の影響？

3. **透明度**:
   - opacity: 0が設定されている？（CSSで）

4. **マーカー未定義**:
   - `#manual-edge-arrowhead` が存在しない？
   - defs要素が正しく作成されていない？

5. **座標値が異常**:
   - NaN や Infinity になっている？

6. **親要素の問題**:
   - SVG要素自体が非表示？
   - transform属性で画面外に？

### ユーザーレポート

> そのエッジとエッジラベルは表示されませんでした。
