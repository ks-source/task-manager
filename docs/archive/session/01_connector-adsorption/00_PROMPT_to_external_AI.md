## 背景

Mermaidで自動生成されたSVGフローチャートに対して、手動でエッジ（矢印線）を追加する機能を実装中です。Phase 2として「ノード自動吸着機能」を実装していますが、2つの問題に直面しています。

## 解決したい問題

### 問題1: サブグラフが個別ノードより優先的に吸着されてしまう

**期待動作**:
- ユーザーが個別ノード（例: PHASE1, PHASE2）付近をクリック
- 最寄りの**個別ノード**に吸着してエッジを接続

**実際の動作**:
- サブグラフ（cluster, 例: SCOPE）が大きいため、その中心までの距離が優先される
- 個別ノードをクリックしたつもりでもサブグラフに吸着してしまう

**技術詳細**:
- `findNearestNode()` 関数がユークリッド距離で最寄りノードを検索
- `[data-et="node"]` と `[data-et="cluster"]` の両方を検索対象にしている
- サブグラフは大きな矩形領域（例: 869.0x803.0）を持つ
- 個別ノードは小さな矩形領域（例: 598.0x858.0）を持つ

**質問**:
1. サブグラフと個別ノードの両方を吸着対象としつつ、個別ノードを優先する方法は？
2. bboxサイズで判定すべきか、ノードタイプで優先度を設定すべきか？
3. より良いアルゴリズムはありますか？

### 問題2: エッジとラベルがSVGに表示されない

**期待動作**:
- ユーザーがエッジ作成完了後、青色の太い破線（5px）が表示される
- ラベルテキストがエッジの中点に表示される

**実際の動作**:
- コンソールログでは `drawEdgeToSvg()` が成功している
- `svg.appendChild(g)` も実行されている
- しかし画面上にエッジが見えない

**技術詳細**:
- SVG要素は正しく取得されている（`✅ SVG element found`）
- エッジデータは正しく生成されている（座標、色、太さ、破線パターン）
- `<g class="manual-edge">` 内に `<line>` と `<text>` を作成
- `marker-end="url(#manual-edge-arrowhead)"` を設定
- Z-index問題？viewBox問題？transform問題？

**質問**:
1. `svg.appendChild()` が成功しているのに要素が見えない原因として考えられることは？
2. SVG座標系のズーム（150%）が影響している可能性は？
3. デバッグ方法として何を確認すべきですか？
