# 外部AI有識者への質問プロンプト

## 📋 背景と目的

フローチャートエディター（flowchart-editor.html）において、以下の課題に直面しています：

### 現在の課題
1. **メモノードと正規ノードの関係性が不明確**
   - メモノード（手動追加したノード）がどの正規ノード（Mermaidで生成）に関連するか、視覚的な矢印のみで判断
   - SVG再読込時に矢印の意味（参照関係）が失われる
   - 人間が属人的に判断するしかない状況

2. **ノード吸着機能の実装困難**
   - 当初、メモエッジを正規ノードに自動吸着させる仕様を計画
   - 座標変換とSVG要素検出の技術的困難により実装難航
   - 代替策として明示的なメタデータ管理を検討

3. **タスク連携情報の永続化不足**
   - タスクとフローチャート要素の連携情報（mockTasks）がSVGに保存されない
   - リロード時に連携情報が失われる

### 提案する解決策
**「参照関係（References）」を明示的なメタデータとして管理**

- 全要素（正規ノード、メモノード、エッジ）に共通の `references` フィールドを追加
- 要素情報パネルで人間が手動入力・管理
- SVGメタデータとして保存・復元
- タスク連携情報も同時にメタデータ化

## ❓ 質問事項

### Q1. データモデルの妥当性
提案する参照関係のデータモデルは適切でしょうか？

```javascript
// 提案するデータ構造
elements[id].references = ["U_START", "node_002"]
manualNodes[id].references = ["node_003"]
taskFlowchartLinks = {
  "0.2": {
    taskName: "アーキテクチャ設計",
    flowchartLinks: { nodes: ["U_START"], edges: [] }
  }
}
```

**懸念点**:
- 一方向参照のみ（双方向ではない）で十分か？
- 参照の意味（種類）を区別すべきか？（例: "関連", "補足", "前提"）
- 文字列配列で十分か、オブジェクト配列にすべきか？

### Q2. UI/UXの使いやすさ
タグ形式 + オートコンプリートのUIは使いやすいでしょうか？

```
┌─ 🔗 参照関係（References） ────────────┐
│ このノードが参照・関連する要素:         │
│ ┌─────────────────────────────────┐   │
│ │ [U_START] × │ [node_002] × │    │   │
│ └─────────────────────────────────┘   │
│ ┌─────────────────────────┬─────────┐ │
│ │ 要素を検索...          │ [+追加]│ │
│ └─────────────────────────┴─────────┘ │
└───────────────────────────────────────┘
```

**代替案の検討**:
- ドロップダウンリスト方式
- ドラッグ&ドロップ方式
- グラフビジュアライザー（ノード関係図）

### Q3. 循環参照の扱い
循環参照（A → B → A）を許可する設計で問題ないでしょうか？

**現在の方針**:
- 許可する（参照は依存関係ではなく「関連性」を示すため）
- 検出は行うが、警告のみでエラーにしない

**懸念点**:
- AI解析時に無限ループのリスクはないか？
- グラフ探索アルゴリズムで問題が生じないか？

### Q4. タスク連携のメタデータ化
タスク連携情報（mockTasks）をSVGメタデータに含めるべきでしょうか？

**現状**:
- タスクマネージャー（A側）とフローチャートエディター（B側）はLocalStorageで連携
- B側のmockTasksはメモリ上のみで、SVGには保存されない

**懸念点**:
- SVGファイルサイズの肥大化
- データの重複管理（LocalStorage vs SVGコメント）
- 同期ずれのリスク

### Q5. 実装の優先順位
Phase 1-3の実装順序は適切でしょうか？

- **Phase 1**: データ構造整備（2-3h）
- **Phase 2**: UI実装（3-4h）
- **Phase 3**: 利便性向上（2-3h、任意）

**代替案**:
- Phase 1でMVP（最小限のUI）を含めるべきか？
- Phase 3を先行して実装すべき機能はあるか？

## 📝 補足情報

### プロジェクト概要
- **ツール名**: Task Manager（タスクマネージャー + フローチャートエディター）
- **構成**: 単一HTMLファイル、外部依存なし
- **技術スタック**: Vanilla JavaScript, File System Access API, LocalStorage
- **対象ユーザー**: プロジェクトマネージャー、システムアーキテクト

### 既存機能
- Mermaid SVGフローチャートの読込・編集
- メモノード・手動エッジの追加
- 要素へのメモ・カスタムラベル付与
- タスクマネージャーとの双方向連携（LocalStorage経由）
