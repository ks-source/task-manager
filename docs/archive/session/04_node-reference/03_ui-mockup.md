## 🖱️ UI実装パターン比較

### パターンA: ドロップダウン + 検索

```
┌──────────────────────────────────────────┐
│ 参照する要素を追加:                      │
│                                          │
│ ┌────────────────────────────┬────────┐ │
│ │ 要素を選択...              │  [▼] │ │
│ └────────────────────────────┴────────┘ │
│   ↓ クリックでドロップダウン展開        │
│ ┌──────────────────────────────────┐   │
│ │ 🔍 node_                        │   │ ← インクリメンタル検索
│ ├──────────────────────────────────┤   │
│ │ ○ U_START - [USR_01] ユーザー   │   │
│ │ ○ node_002 - [SYS_01] システム  │   │
│ │ ○ node_003 - [SYS_02] DB        │   │
│ │ ○ MEMO_001 - 重要な注意事項      │   │
│ │ ○ E_MANUAL_001 - 補足エッジ      │   │
│ └──────────────────────────────────┘   │
│                                          │
│ 選択された要素:                          │
│ ┌──────────────────────────────────┐   │
│ │ [U_START] ×  [node_002] ×        │   │
│ └──────────────────────────────────┘   │
└──────────────────────────────────────────┘
```

**利点**:
- 既存のUIパターンで理解しやすい
- 全候補を一覧表示

**欠点**:
- クリック操作が多い
- モバイルで扱いにくい

### パターンB: タグ入力形式（推奨）

```
┌──────────────────────────────────────────┐
│ 参照する要素を追加:                      │
│                                          │
│ ┌──────────────────────────────────┐   │
│ │ [U_START] ×  [node_002] ×        │   │ ← 既存タグ（×で削除）
│ │ ┌──────────────────────────┐    │   │
│ │ │ 要素IDまたは名前を入力... │    │   │ ← 入力フィールド
│ │ └──────────────────────────┘    │   │
│ └──────────────────────────────────┘   │
│   ↓ 入力中                              │
│ ┌──────────────────────────────────┐   │
│ │ 候補:                            │   │ ← オートコンプリート
│ │ ○ U_START - [USR_01] ユーザー   │   │
│ │ ○ node_002 - [SYS_01] システム  │   │
│ │ ○ node_003 - [SYS_02] DB        │   │
│ └──────────────────────────────────┘   │
│                                          │
│ 💡 ヒント: 要素IDの一部を入力すると     │
│    候補が表示されます                   │
└──────────────────────────────────────────┘
```

**利点**:
- キーボード操作のみで完結
- 複数要素を素早く追加可能
- モダンなUX（タグ入力はGmail等で採用）

**欠点**:
- 要素IDを知らないと使いにくい（オートコンプリートで軽減）

**推奨理由**:
- 効率性が高い
- 既存のタグ入力UIライブラリを参考にできる
- フローチャート編集という専門的な用途に適している

## 🔄 参照関係の編集フロー

### 追加操作

```
1. 要素を選択（クリック）
┌─────────────────────────┐
│   ┌─────────┐           │
│   │ U_START │ ← クリック │
│   └─────────┘           │
└─────────────────────────┘
        ↓
2. 要素情報パネルが開く
┌─────────────────────────┐
│ 📍 要素情報 - U_START   │
│ ▶ 🔗 参照関係 (0件)     │
└─────────────────────────┘
        ↓ クリックで展開
3. 参照関係セクションを展開
┌─────────────────────────┐
│ ▼ 🔗 参照関係 (0件)     │
│ ┌───────────────────┐   │
│ │ （参照なし）      │   │
│ │ ┌─────────────┐  │   │
│ │ │ node_       │  │   │ ← 入力開始
│ │ └─────────────┘  │   │
│ └───────────────────┘   │
└─────────────────────────┘
        ↓ 入力中
4. オートコンプリート表示
┌─────────────────────────┐
│ │ node_002        │  │   │
│ └─────────────┘  │   │
│ 候補:              │   │
│ ○ node_002 - [SYS_01] │ ← クリックで選択
│ ○ node_003 - [SYS_02] │
└─────────────────────────┘
        ↓ 選択
5. タグとして追加
┌─────────────────────────┐
│ ▼ 🔗 参照関係 (1件)     │
│ ┌───────────────────┐   │
│ │ [node_002] ×      │   │ ← 追加完了
│ │ ┌─────────────┐  │   │
│ │ │             │  │   │
│ │ └─────────────┘  │   │
│ └───────────────────┘   │
└─────────────────────────┘
        ↓ 自動保存
6. SVGメタデータに保存
```

### 削除操作

```
1. 削除したい参照のタグをクリック
┌─────────────────────────┐
│ [node_002] × ← クリック  │
│ [MEMO_001] ×            │
└─────────────────────────┘
        ↓
2. 確認ダイアログ（任意）
┌─────────────────────────┐
│ 参照を削除しますか？    │
│                         │
│ node_002 - [SYS_01]     │
│                         │
│ [キャンセル] [削除]     │
└─────────────────────────┘
        ↓ 削除クリック
3. タグが削除される
┌─────────────────────────┐
│ [MEMO_001] ×            │
│ （空欄）                │
└─────────────────────────┘
        ↓ 自動保存
4. SVGメタデータから削除
```
