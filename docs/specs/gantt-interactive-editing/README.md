## 概要

ガントチャート専用画面において、タスクの日程バーを**直接ドラッグ&リサイズ**することで、視覚的かつ直感的に日程調整を行える機能。変更内容は即座にメイン画面のタスクデータへ反映される。

### 背景

現在のガントチャート機能（v2.8.0以降）は、タスクの日程を**読み取り専用**で表示するのみ。日程調整が必要な場合は、メイン画面のタスク詳細モーダルで日付入力が必要で、ワークフローが分断されている。

### 目的

1. **直感的な日程調整**: ドラッグ&ドロップでタスク日程を変更
2. **即時反映**: 変更内容がメイン画面へリアルタイム同期
3. **視覚的フィードバック**: プレビュー表示で変更結果を事前確認
4. **柔軟な編集**: 開始日・終了日・期間の個別調整に対応

## 主要機能

### ✅ 実装する機能

| 機能 | 説明 | 優先度 |
|------|------|--------|
| **日程バー左端リサイズ** | 開始日を変更（終了日固定） | 🔴 必須 |
| **日程バー右端リサイズ** | 終了日を変更（開始日固定） | 🔴 必須 |
| **日程バー中央ドラッグ** | 期間維持したまま移動 | 🔴 必須 |
| **リアルタイムプレビュー** | ドラッグ中に変更後の日程を表示 | 🟡 高 |
| **メイン画面への即時反映** | mouseup時に自動同期 | 🔴 必須 |
| **制約チェック** | 最低1日期間、先行タスク依存関係の警告 | 🟡 高 |

### ❌ 実装しない機能（明示的除外）

| 機能 | 実装しない理由 |
|------|--------------|
| **Undo/Redo** | 実装コスト高、Phase 2以降で検討 |
| **先行タスク自動調整** | 複雑すぎる、警告表示のみ |
| **複数タスク同時選択** | 初期実装では単一タスク編集に集中 |
| **タッチデバイス対応** | Edge/Chromeデスクトップ版を想定 |

## 技術スタック

- **言語**: JavaScript（ES6+）
- **既存機能**: ガントチャート表示機能（v2.8.0以降）
- **ウィンドウ間通信**: `window.opener` API
- **対象ブラウザ**: Edge/Chrome 90+
- **主要API**:
  - Mouse Events (mousedown, mousemove, mouseup)
  - getBoundingClientRect()
  - File System Access API（既存の保存機能）

## 開発フェーズ

### Phase 1: 基本ドラッグ移動（6-8h）
- 日程バーのmousedown/move/upイベント
- 中央ドラッグによる移動（期間維持）
- メイン画面への反映（`updateTaskDates`関数追加）

### Phase 2: リサイズ機能（5-7h）
- 左端/右端のリサイズハンドル判定
- 開始日/終了日の個別変更
- 最小期間（1日）の制約

### Phase 3: UX改善（4-6h）
- プレビュー表示
- カーソル変化
- エラー警告表示
- テスト・デバッグ

**合計工数**: 15-21h（2-3日間）

## データフロー概要

```
ガントチャート画面               メイン画面 (task-manager.html)
┌────────────────┐              ┌────────────────┐
│ ドラッグ開始    │              │                │
│  ↓             │              │                │
│ mousemove中    │              │                │
│ ・プレビュー表示│              │                │
│ ・新日付計算    │              │                │
│  ↓             │              │                │
│ ドラッグ終了    │─────通信───►│ window.opener  │
│ mouseup        │   (1)       │ .updateTaskDates()│
│                │              │  ↓             │
│                │              │ タスクデータ更新│
│                │              │ start_date     │
│                │              │ finish_date    │
│                │              │  ↓             │
│                │◄────通知─────│ 更新完了       │
│ 確認通知表示    │   (2)       │                │
└────────────────┘              └────────────────┘

通信データ例:
window.opener.updateTaskDates(
  "WBS1.1.0",      // タスクID
  "2026-03-04",    // 新開始日
  "2026-03-08"     // 新終了日
);
```

## リスクと対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| 表示モード（Day/Week/Month）での座標計算の複雑化 | 中 | モード別の変換関数を分離実装 |
| 先行タスク依存関係の整合性 | 高 | 変更時に警告表示、強制禁止はしない |
| ガントチャート画面とメイン画面の同期タイミング | 中 | mouseup時に即座に反映、5秒自動更新は継続 |
| 営業日計算の精度 | 低 | 既存の`calculateBusinessDays`関数を流用 |
